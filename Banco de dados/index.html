Estrutura de dados

Estruturas de dados organizam e armazenam dados eficientemente, facilitando o acesso e a manipulação. Exemplos incluem arrays, listas, pilhas, filas, árvores e grafos.


  • Dados Estruturados
  • Dados Não Estruturados
  • Dados Semi Estruturados

    Banco de Dados Relacionais

    Bancos de dados relacionais armazenam dados em tabelas organizadas em linhas e colunas, permitindo a relação entre tabelas através de chaves primárias e estrangeiras. Eles usam SQL para gerenciar e consultar dados de forma estruturada e eficiente.


  • PostgreSQL
  • MySQL
  • Oracle Database
  • Microsoft SQL Server

    • 3

    Modelo Entidade Relacionamento

    O Modelo Entidade-Relacionamento (ER) é uma metodologia para modelar dados através de entidades, atributos e relacionamentos, criando uma representação visual do esquema do banco de dados.


  • Entidade Forte
  • Entidade Fraca
  • Relacionamento

    • 4

    Primary key e Foreign key

    Primary key é um atributo que identifica de forma única cada registro em uma tabela de um banco de dados. Foreign key é um atributo que estabelece uma relação entre duas tabelas, referenciando a primary key de outra tabela.


  • Primary Key(Chave Primária):
  • Deve conter valores únicos
  • Não pode conter valores nulos
  • Foreign Key (Chave Estrangeira):
  • Deve referenciar uma chave primária

    DDL - Data Definition Language

    DDL (Data Definition Language) é uma linguagem usada para definir a estrutura e características dos objetos de um banco de dados, como tabelas, índices e restrições. Ela permite criar, modificar e excluir esses objetos, definindo suas propriedades e relacionamentos.


  • CREATE
  • ALTER
  • DROP
  • TRUNCATE
  • COMMENT
  • RENAME

    • 6

    DML - Data Manipulation Language

    DML (Data Manipulation Language) é uma linguagem de programação que permite aos usuários acessar e manipular dados em bancos de dados, fornecendo meios para executar operações como inserção, atualização, exclusão e recuperação de dados.


  • INSERT
  • UPDATE
  • DELETE
  • SELECT

    • 7

    DQL - Data Query Language

    DQL (Data Query Language) é uma linguagem de consulta usada para extrair informações específicas de um banco de dados, utilizando principalmente o comando SELECT. Essencial para análises e relatórios, permite filtrar, ordenar e agrupar dados de maneira eficiente.


  • SELECT
  • FROM
  • WHERE
  • ORDER BY
  • JOIN

    • 8

    DTL - Data Trasaction Language

    DTL (Data Transaction Language) é uma linguagem usada para gerenciar transações em um banco de dados, permitindo garantir a consistência e integridade dos dados. Inclui comandos como COMMIT para confirmar transações e ROLLBACK para desfazer alterações em caso de erro.


  • COMMIT
  • ROLLBACK
  • BEGIN TRANSACTION

    • 9

    DCL - Data Control Language

    DCL (Data Control Language) é uma linguagem usada para gerenciar privilégios de acesso e segurança em um banco de dados, garantindo a proteção dos dados e o controle adequado sobre quem pode visualizar ou modificar informações.


  • GRANT
  • REVOKE
  • DENY

    • 10

    Banco de Dados NoSQL

    Bancos de Dados NoSQL são sistemas de armazenamento que oferecem flexibilidade para estruturar e manipular dados não relacionais, ideal para cenários de grande volume, variedade ou velocidade de dados, com modelos de dados que variam de documentos a grafos.


  • Documentos
  • Chave-Valor
  • Colunas
  • Grafos
