As estruturas de decisão na programação permitem que um programa selecione diferentes caminhos de execução com base em condições específicas.


  • if
  • else
  • else if

    Estrutura de Repetição

    A estrutura de repetição é um conceito fundamental na programação, permitindo que um bloco de código seja executado repetidamente enquanto uma condição específica for verdadeira.


  • while
  • do while
  • for

    Vetores

    Vetores, ou arrays, são estruturas de dados que armazenam coleções ordenadas de elementos do mesmo tipo. Eles permitem o acesso eficiente aos elementos por meio de índices e são amplamente utilizados para armazenar e manipular conjuntos de dados em programação.


  • Array de Inteiros
  • Array de Strings
  • Array de Booleanos
  • Array de Objetos

    Matriz

    Uma matriz é uma estrutura de dados que armazena uma coleção de elementos em uma grade bidimensional.


  • Array de Inteiros
  • Array de Strings
  • Array de Booleanos
  • Array de Objetos

    Funções

    Funções em programação encapsulam blocos de código para reutilização, melhorando modularidade e legibilidade do código, além de permitir parâmetros e retorno de valores.


  • Funções sem retorno(void)
  • Funções com retorno
  • Funções recursivas

    Programação Orientada à Objeto

    Programação Orientada a Objetos (POO) é um paradigma de programação que organiza o código em objetos que combinam dados e comportamentos, facilitando a reutilização, modularidade e manutenção do código.


  • Atributos
  • Construtor
  • Métodos
  • Getters & Setters
  • toString

    • 7

    Encapsulamento

    Encapsulamento em Programação Orientada a Objetos (POO) protege os detalhes internos de uma classe, permitindo acesso controlado aos seus membros e promovendo a segurança e integridade dos dados.


  • Private
  • Public
  • protected

    • 8

    Herança

    Herança em POO permite que uma classe (subclasse ou filho) herde atributos e métodos de outra classe (superclasse ou pai), promovendo reutilização de código e estabelecendo relações hierárquicas entre classes.

    9

    Sobrecarga

    Sobrecarga em POO permite que uma classe tenha múltiplos métodos com o mesmo nome, mas com diferentes tipos ou quantidade de parâmetros, permitindo comportamentos distintos para o mesmo método.


  • Sobrecarga de método
  • Sobrecarga de operador

    • 10

    Classes Estáticas

    Classes estáticas são classes que não podem ser instanciadas e cujos membros são acessados diretamente pela classe, sem a necessidade de criar um objeto. Elas são usadas para agrupar métodos e propriedades que não dependem de instâncias específicas.

    11

    Exceções

    Exceções são eventos que ocorrem durante a execução de um programa, interrompendo o fluxo normal de instruções, e são usadas para tratar erros e situações inesperadas de maneira controlada. Elas são manipuladas com blocos de código específicos como try, catch e finally.

    12

    Classes e métodos abstratos

    Classes abstratas servem como modelos para outras classes, definindo a estrutura e funcionalidades básicas que as classes derivadas devem implementar. Elas não podem ser instanciadas diretamente. Já os métodos abstratos são métodos sem implementação completa, definindo apenas a assinatura e obrigando as classes derivadas a implementá-los de acordo com suas necessidades específicas.

    13

    Polimorfismo

    Polimorfismo é a capacidade de um objeto assumir diversas formas e responder a mensagens de diferentes maneiras, de acordo com seu tipo real. Isso promove flexibilidade e reutilização de código na programação orientada a objetos.
